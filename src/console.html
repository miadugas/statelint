How to use statelint

        

          

            
1This console

            
You're looking at it — served locally from your repo:

            

              statelint --ui src/
            

            

              Edit code, then hit ⟳ Rescan — the analysis re-runs
              in-process (~1s on a mid-size app) and every panel updates. No
              file juggling, no re-running the CLI. Add --port N to
              change the port (default 8734).
            

          


          

            
2Read a finding

            

              

                file line:col
                
                  click it — opens the file in VS Code scrolled to the exact
                  line (vscode:// deep link)
                
              

              

                severity
                
                  High = error · Medium = warn (fails CI) · Low = info
                  (advisory)
                
              

              

                rule
                which detector fired
              

              

                → fix
                
                  the named refactor — stack-aware: names the tool
                  your app already uses most
                
              

            

          


          

            
3What it understands

            
Ten state surfaces plus memoization, one graph:

            

              

                client
                
                  useState/useReducer · Context · Zustand · Redux/RTK slices
                
              

              

                server cache
                TanStack Query · RTK Query
              

              

                persistence
                localStorage · sessionStorage · cookies
              

              

                address bar
                URL search params · route params · nuqs
              

              

                memoization
                React.memo · useMemo · useCallback
              

            

          


          

            
4The rules

            

              

                multiple-sources-of-truth
                
                  one entity owned by 2+ global sources — store vs context vs
                  storage vs URL vs cookie — or server data cached twice
                
              

              

                server-state-in-client-state
                
                  fetch + useState + useEffect hand-rolled cache, grouped per
                  effect; prefilled form drafts soften to Low
                
              

              

                derived-state-as-state
                
                  useState recomputed from other state by a sync effect — should
                  be a useMemo
                
              

              

                storage-as-state
                
                  localStorage/sessionStorage key shared as a non-reactive store
                
              

              

                cookie-as-state
                
                  cookie shared via js-cookie/document.cookie across components
                
              

              

                url-state-forked
                
                  useState copies of search params that go stale on back/forward
                
              

              

                prop-drilling
                props through components that never read them
              

              

                over-globalized-state
                global store/context/slice with one real consumer
              

              

                over-broad-selector
                whole-store subscriptions re-rendering on every change
              

              

                defeated-memo
                
                  React.memo receiving inline props — the memo never holds
                
              

              

                pointless-memo
                useMemo/useCallback with no deps or inline-literal deps
              

            

          


          

            
5Command line & CI

            
Same engine, headless — for pre-commit hooks and pipelines:

            

              statelint src/
            

            

              

                --json
                machine-readable output
              

              

                --min-drill N
                prop-drilling threshold (default 2 blind hops)
              

              

                --no-color
                plain text; also honors NO_COLOR / pipes
              

            

          


          

            
6Exit codes

            

              

                exit 0
                clean, or Low (advisory) findings only
              

              

                exit 1
                Medium/High findings — fails CI on purpose
              

              

                exit 2
                usage error (bad flag, no files found)
              

            

            

              Test files (*.test.*, __tests__,
              stories) are skipped automatically — rules describe the app, not
              its tests.
